# Multi-Project Dashboard Plan

## Design Philosophy

**Goal:** Help investors answer "How is my portfolio performing?" within 5 seconds of looking at the dashboard.

**Principles:**
1. **Clarity over comprehensiveness** - Show what matters, not everything possible
2. **Actionable insights** - Highlight items requiring attention
3. **Visual hierarchy** - Most important metrics largest/highest
4. **Progressive disclosure** - Summary first, drill-down available
5. **Respect existing patterns** - Use established 4-column layouts and color coding

---

## Dashboard Structure

### Primary Sections (in order of importance)

1. **Portfolio Health Summary** - The "5-second answer"
2. **Active Projects Pipeline** - What needs attention now
3. **Financial Performance** - ROI and profit tracking
4. **Budget Insights** - Where is money going?

## Data Requirements

### New Database View: `portfolio_summary`

```sql
CREATE VIEW portfolio_summary AS
SELECT
  -- Total counts
  COUNT(*) as total_projects,
  COUNT(*) FILTER (WHERE status NOT IN ('sold', 'dead')) as active_projects,
  COUNT(*) FILTER (WHERE status = 'analyzing') as analyzing_count,
  COUNT(*) FILTER (WHERE status = 'under_contract') as under_contract_count,
  COUNT(*) FILTER (WHERE status = 'in_rehab') as in_rehab_count,
  COUNT(*) FILTER (WHERE status = 'listed') as listed_count,
  COUNT(*) FILTER (WHERE status = 'sold') as sold_count,

  -- Financial aggregates
  SUM(arv) FILTER (WHERE status NOT IN ('sold', 'dead')) as total_arv,
  SUM(total_investment) FILTER (WHERE status NOT IN ('sold', 'dead')) as capital_deployed,
  SUM(gross_profit) FILTER (WHERE status = 'sold') as total_profit,
  AVG(roi) FILTER (WHERE status = 'sold') as avg_roi,

  -- Budget aggregates
  SUM(rehab_budget) as total_budget,
  SUM(rehab_actual) as total_actual,

  -- Risk indicators
  COUNT(*) FILTER (WHERE rehab_actual > rehab_budget) as over_budget_count,
  COUNT(*) FILTER (WHERE target_complete_date < NOW() AND status = 'in_rehab') as behind_schedule_count

FROM project_summary
WHERE user_id = auth.uid();
```

### New Database View: `category_totals`

```sql
CREATE VIEW category_totals AS
SELECT
  category,
  SUM(budget) as total_budget,
  SUM(actual) as total_actual,
  SUM(actual) - SUM(budget) as variance,
  COUNT(*) as item_count
FROM budget_items bi
JOIN projects p ON bi.project_id = p.id
WHERE p.status NOT IN ('sold', 'dead')
GROUP BY category
ORDER BY total_actual DESC;
```

## What We're NOT Including (Intentionally)

To keep the dashboard focused and not overwhelming:

1. **No vendor performance section** - Valuable but secondary; accessible from Vendors tab
2. **No detailed draw/payment tracking** - Project-level concern, not portfolio
3. **No timeline/Gantt charts** - Complex, better as separate feature
4. **No market data/trends** - Out of scope for MVP dashboard
5. **No comparison to previous periods** - Add later based on user feedback
6. **No export/reporting** - Future enhancement
